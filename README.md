# SHL Conversational Assessment Recommender

A conversational AI-powered assessment recommendation system that helps recruiters identify the most relevant SHL assessments through natural language conversations.

The system accepts hiring requirements, asks clarifying questions when needed, recommends appropriate SHL assessments, refines recommendations as user requirements change, and compares assessments using only grounded evidence from the official SHL catalog.

---

# Features

- Conversational recommendation workflow
- Clarifying questions for ambiguous requests
- Context-aware recommendation refinement
- Assessment comparison
- Grounded recommendations using the SHL catalog
- Prompt injection and off-topic request handling
- REST API built with FastAPI
- Interactive Swagger documentation
- Hybrid-ready retrieval architecture
- Stateless conversation handling

# Retrieval Pipeline

The recommendation engine retrieves assessments from the SHL catalog using a lightweight information retrieval pipeline.

The catalog currently contains **377 SHL Individual Assessments**.

Each assessment indexes:

- Assessment Name
- Description
- Assessment Type
- Job Levels
- Languages
- Duration
- Remote Availability
- Adaptive Testing Support

The retrieval engine performs:

1. Query preprocessing
2. Tokenization
3. TF-IDF vector generation
4. Cosine similarity ranking
5. Optional semantic indexing support for future hybrid retrieval

Recommendations are always selected from the SHL catalog only.

No assessment names or URLs are generated by the model.

# Conversation Flow

The assistant supports four primary interaction types.

### 1. Clarification

When insufficient information is provided, the assistant asks follow-up questions.

Example:

> "I'm hiring."

↓

"What role are you hiring for?
Are you looking for technical, cognitive or personality assessments?"

---

### 2. Recommendation

Once enough context is available, the assistant recommends relevant SHL assessments.

Example:

> Java Backend Developer

↓

Returns:

- Java 8
- Core Java (Advanced)
- Java EE 7
- Java Frameworks

---

### 3. Refinement

The assistant supports iterative refinement.

Example:

User:

> Recommend assessments for Java Developers.

Later:

> Add personality assessments under 30 minutes.

Recommendations are regenerated using the updated constraints.

---

### 4. Comparison

Users can compare two SHL assessments.

Comparison includes:

- Purpose
- Assessment Type
- Duration
- Job Levels
- Remote Support
- Adaptive Testing

Only information from the SHL catalog is used.

## Chat Endpoint

```
POST /chat
```

Example Request

```json
{
  "messages": [
    {
      "role":"user",
      "content":"Recommend assessments for a Java Backend Developer."
    }
  ]
}
```

Example Response

```json
{
  "reply":"Here are some recommended SHL assessments...",
  "recommendations":[
    {
      "name":"Java 8",
      "url":"https://www.shl.com/...",
      "test_type":"Knowledge & Skills"
    }
  ],
  "end_of_conversation":false
}
```

# Evaluation

The recommender was evaluated using representative conversational scenarios.

| Scenario | Expected Behaviour |
|-----------|-------------------|
| Vague hiring request | Clarifying question |
| Technical hiring request | Relevant recommendations |
| Follow-up refinement | Updated recommendation list |
| Assessment comparison | Grounded comparison |
| Prompt injection | Refusal |

Evaluation focused on:

- Retrieval Quality
- Recommendation Relevance
